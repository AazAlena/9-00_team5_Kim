# 🩺 Веб-приложение «График приёмов врача» (Кейс C07)

Информационная система автоматизации расписания, генерации временных слотов и записи пациентов, реализованная на чистом веб-стеке без сторонних API. Система включает аналитический модуль расчёта утилизации рабочего времени медицинского персонала с выгрузкой отчётов в Excel.

---

## 📊 1. Бизнес-цель и KPI

* **Бизнес-цель:** Оптимизировать загрузку медицинского персонала клиники, минимизировать количество простоев («окон») в расписании и исключить ручные ошибки при бронировании времени.
* **Ключевой KPI:** Коэффициент утилизации (загрузки) расписания врача (*Utilization*).
* **Формула расчета:** 
  \[\text{Utilization} = \frac{\text{slots\_booked}}{\text{slots\_total}} \times 100\%\]
* **Аналитические срезы:** Фильтрация на дашборде по конкретному врачу, специальности и временному диапазону (день/неделя).

---

## 🔒 2. Безопасность и ограничения MVP

* **Изолированный контур:** Приложение функционирует автономно. Внешние сетевые API и интеграции (включая 1С и ЕМИАС) отсутствуют. Связность бэкенда и фронтенда реализована напрямую через JavaScript.
* **Защита ПДн:** Обработка реальных персональных данных не производится. Система заполнена синтетическими обезличенными тест-наборами (T1/Т2).
* **Ограничение интерфейса:** Интерфейс спроектирован и оптимизирован строго под десктопные экраны (адаптивная верстка под мобильные устройства в рамках MVP не предусмотрена).

---

## 🛠 4. Бизнес-правила и валидаторы (JavaScript)

В приложение зашиты следующие жёсткие контроли данных:
1. **Запрет двойного бронирования (AC-03):** JS-валидатор блокирует запись, если на выбранный `slot_id` уже создана активная бронь.
2. **Логирование отмен:** При переводе записи в статус `canceled` скрипт автоматически требует ввод причины и логирует её в сущность `canceled_appointment`.
3. **Контроль дубликатов:** При повторном чтении исходных файлов система дедуплицирует записи по первичным ключам.

---

## 🎬 5. Пошаговый сценарий проверки (Инструкция к Live Demo)

Для демонстрации работоспособности системы экспертной комиссии необходимо выполнить шаги сценария **D7**:

1. **Шаг 1 (Авторизация):** Войдите в систему под соответствующей учётной записью (Пациент, Врач или Администратор).
2. **Шаг 2 (Тестирование импорта):** Выполните загрузку исходных файлов из каталога `/data/C07/T1/`. Интерфейс отобразит календарную сетку слотов.
3. **Шаг 3 (Проверка валидатора):** Попробуйте перезаписать пациента на уже занятое время. Система должна вывести ошибку бизнес-правила.
4. **Шаг 4 (Целевое действие):** Создайте 2 новые успешные записи на свободные слоты и выполните 1 отмену существующей записи с указанием причины.
5. **Шаг 5 (Аналитика и Выходной контроль):** Откройте личный кабинет администратора, примените фильтр по врачу и нажмите кнопку «Экспорт». Система мгновенно сформирует и скачает аналитический отчёт в формате **Excel (.xlsx)**.
