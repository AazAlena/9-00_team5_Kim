# Генерация набора данных T2 для Кейса 7

**ID кейса:** C07
**Название:** График приёмов врача
**SEED (начальное число):** 2025
**Инструмент генерации:** SQLite 3

## Объёмы данных (T2)

| Таблица | Количество строк в T2 |
| :--- | :---: |
| `speciality.csv` | 40 |
| `user.csv` (врачи) | 40 |
| `user.csv` (пациенты) | 180 |
| `work_slot.csv` | ~1200 |
| `appointment.csv` | ~2000 |
| `canceled_appointment.csv` | ~300 |

**Примечание:** T2 примерно в 10 раз больше T1, как требуется в Приложении С.

## Правила генерации

### 1. speciality.csv
- Каждому врачу случайным образом назначается одна специальность
- Список специальностей: `Терапевт`, `Хирург`, `Дерматолог`, `Окулист`, `ЛОР`, `Невролог`, `Кардиолог`

### 2. user.csv

**Врачи (40 человек):**
- `id`: `doc_001` ... `doc_040`
- `role`: `doctor`
- ФИО: случайная комбинация из списков фамилий, имён и отчеств
- Email: `doc_XXX@clinic.com`

**Пациенты (180 человек):**
- `id`: `pat_0001` ... `pat_0180`
- `role`: `patient`
- ФИО: случайная комбинация из списков фамилий и имён
- Email: `pat_XXXX@mail.com`

**Используемые списки:**
- Фамилии: Иванов, Петров, Сидоров, Кузнецова, Смирнова, Волков, Морозова, Новиков, Федорова, Васильев
- Имена: Иван, Пётр, Сидор, Анна, Мария, Елена, Дмитрий, Ольга, Алексей, Татьяна
- Отчества: Ивановна, Петрович, Сергеевна, Андреевна, Владимирович

### 3. work_slot.csv
- Период: 30 дней, начиная с `2024-10-21`
- Каждый врач получает по одной записи на каждый день
- Начало работы: случайно (8:00, 9:00 или 10:00)
- Продолжительность рабочего дня: 8–10 часов
- Длительность слота: случайно (15, 20, 30 или 60 минут)
- Перерыв: 30 или 60 минут, примерно в середине дня

### 4. appointment.csv
- Для каждого `work_slot` генерируется 1–3 записи
- Общее количество записей: ~2000
- Распределение статусов:
  - `booked` — 60%
  - `served` — 30%
  - `canceled` — 10%
- Записи распределяются случайным образом по доступным слотам
- Нет дубликатов: один врач не может иметь две записи на одно и то же время

### 5. canceled_appointment.csv
- Создаётся для каждой записи со статусом `canceled`
- Причина отмены выбирается случайно из списка:
  - «Передумал(а)»
  - «Не пришёл(ла)»
  - «Ошибка записи»
  - «Перенос»

## Соблюдение бизнес-правил

| Правило | Проверка |
| :--- | :--- |
| Нет пересекающихся слотов у одного врача | Каждый врач имеет одну запись в `work_slot` на день |
| Время записи в пределах рабочего времени | `slot_datetime` попадает в интервал `start_time`–`end_time` |
| Нет двойного бронирования | Уникальность пары `(doctor_id, slot_datetime)` |
| Внешние ключи корректны | `doctor_id` и `patient_code` существуют в `user.csv` |
| Статусы допустимы | Только `booked`, `served`, `canceled` |

## Как воспроизвести генерацию

```bash
# 1. Убедитесь, что SQLite установлен

# 2. Запустите скрипт генерации
sqlite3 :memory: < tools/generate_t2.sql

# 3. Проверьте результат валидатором
python tools/validate.py --path ./data/C07/T2/
